Testes: TDD, Pytest

TEST-DRIVEN DEVELOPMENT

TDD é a sigla para Desenvolvimento Orientado por Testes. É um processo usado antes ou durante o desenvolvimento de novos softwares. Seu objetivo principal é realizar um ciclo de repetições, enquanto o desenvolvedor do programa registra testes automatizados para validar requisitos e implementar novas funcionalidades. Com o TDD, os testes são feitos antes do sistema ser codificado. A técnica auxilia no entendimento do negócio do software, deixando claras as regras antes mesmo de iniciar a codificação, obtendo assim, simplicidade e facilidade.

FASES DO TDD

  • Fase vermelha: escrever um teste.
  • Fase verde: fazer o teste passar, escrevendo o código que ele protege.
  • Fase azul: refatorar.

    • VANTAGENS:

  • O código será mais limpo e simples;
  • Menos bugs no produto;
  • Correção mais completa dos bugs encontrados;
  • A confiabilidade do código será maior;
  • Diminuição de tempo com depuração de código;
  • Favorece o processo de documentação;
  • Torna a Refactoring mais fácil.

    • POR QUE É TÃO DIFÍCIL EXERCER O TDD?

  • Porque você tem que pensar no que quer que seu código realize, e como protegê-lo contra falhas;
  • Porque ele possui uma curva de aprendizagem íngreme. Você precisa aprender os princípios e design patterns para criar um código limpo e como refatorá-lo para mantê-lo dessa forma.

    • ERROS MAIS COMUNS

  • Não seguir a abordagem de test-first;
  • Não refatorar o tempo todo;
  • Escrever mais de um teste de cada vez;
  • Não realizar testes com frequência, perdendo o feedback antecipado deles;
  • Escrever testes que são lentos.

    • PYTEST

    O pytest é um framework de teste para python que promove soluções para fazer testes e realizar validações, facilita a escrita de testes pequenos e legíveis e pode ser dimensionada para dar suporte a testes funcionais complexos para aplicativos e bibliotecas.

    CARACTERÍSTICAS

  • Descoberta automática de módulos e funções de teste;
  • Acessórios modulares para gerenciar recursos de teste pequenos ou parametrizados de longa duração;
  • Arquitetura de plugins rica, com mais de 800 plugins externos e comunidade próspera.

    • COMANDOS

  • Para instalar o Pytest, você deve digitar a seguinte instrução no Prompt de Comando: pip install pytest
  • Para executar o teste, utilize o seguinte comando no Prompt: pytest test_iniciante.py
  • Se o arquivo possuir mais de um teste execute esse comando: pytest test_iniciante.py -v
