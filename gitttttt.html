CONTROLE DE VERSÃO: GIT E GITHUB

O QUE É CONTROLE DE VERSÃO?

É o ato de rastrear e administrar as alterações em um código de software. Os sistemas de controle de versão são utensílios de software que auxiliam as equipes a gerenciar as mudanças no código-fonte ao longo do tempo. O software de controle de versão mantém registro de todas as alterações no código em um tipo especial de banco de dados. Em caso de erros, os programadores podem voltar no tempo comparar e verificar versões anteriores do código para ajudar a reparar o erro enquanto diminuem interrupções para todos os membros da equipe.

COMO FUNCIONA?

O conjunto de dado dos programas ficam estocados em um repositório e o histórico de suas versões é salvo nele. Os desenvolvedores podem conectar,

acessar e resgatar a última versão disponível do projeto e efetuar uma cópia local, na qual poderão trabalhar em cima dela e continuar com a

sequência de desenvolvimento. A cada mudança realizada, torna-se possível enviar novamente ao servidor e atualizar a sua versão a partir

outras feitas pelos demais desenvolvedores.

O QUE É GIT?

O Git registra as mudanças que acontecem no código-fonte de um programa. Dessa forma,

é permitido que os dados sejam alterados de forma simultânea, por vários indivíduos,

sem a preocupação que essas mudanças sejam sobrescritas umas pelas outras.

  • - CLONE DO PROJETO

    • Em primeiro lugar devemos duplicar o código-fonte do projeto em nossas máquinas,

    para que os arquivos sejam disponibilizado localmente.



  • - CRIAÇÃO DA BRANCH

    • Criação de uma nova ramificação para que seja possível podermos alterar os arquivos do projeto sem interferir nos originais.



  • - COMMITS

    • Conforme vão sendo criados e alterados os arquivos, elas vão sendo divididas em commits.



  • - PUSH

    • Uma vez que a funcionalidade está finalizada, enviamos nossas Branchs, com todas as modificações de volta ao repositório

    remoto. Dessa maneira, ela ficará disponível para todos os participantes do projeto poderem ver e alterar os dados.



  • - MERGE

    • Para combinar as alterações com os arquivos originais do projeto da Branch principal ou máster,

    podemos utilizar o comando Merge. Logo em seguida, torna-se crucial dar um commit e um push, para encaminhar

    a ramificação máster mesclada ao repositório remoto e deixar tudo acessível para os contribuintes do projeto.



    COMANDOS DO GIT

  • Init : Inicializar um repositório;
  • Clone: Usado para clonar o repositório;
  • Add: Adicionar um arquivo para o State Area, para que seja vinculado a um próximo commit;
  • Commit: Comando utilizado para mover os arquivos do State Area para o repositório local;
  • Pull: Utilizado para buscar e trazer mudanças do repositório remoto para o repositório local;
  • Push: Utilizado para enviar o arquivo do repositório local para o remoto;
  • Murge: Serve para mesclar commits e Branchs na Branch atual.

    • GITHUB

    GitHub é uma plataforma para administrar códigos e gerar um local de colaboração entre devs, utilizando o Git

    como sistema de controle. O GitHub é mundialmente usado e chega a ter mais de 36 milhões de usuários ativos mundialmente

    contribuindo em projetos comerciais ou pessoais. A plataforma foi idealizada pelo mesmo criador do sistema Linux, Linus Torvalds,

    e se consolidou como uma das mais importantes ferramentas para o desenvolvimento de projetos.

    COMO FUNCIONA O GITHUB?

    O GitHub é um serviço que fica dentro da plataforma Git, funcionando como um gerenciador de projetos em nuvem, facilitando um trabalho mais coordenado, organizado e sistematizado.

    VANTAGENS DO GITHUB

  • Possibilidade de acompanhar e colaborar com projetos de diferentes equipes;
  • Aprender programação na prática;
  • Participar de discussões a respeito de novas tecnologias;
  • Obter auxílio de outros programadores para resolver problemas relacionados a seus projetos;
  • Controlar as diferentes versões de um código com armazenamento em nuvem;
  • Registrar ações e projetos desenvolvidos por você em uma espécie de portfólio online, etc.

    • QUAL A DIFERENÇA DO GIT E O GITHUB?

  • O Git é um software de controle de versão distribuído.
  • O GitHub atua como uma espécie de rede social entre os programadores que podem compartilhar seus códigos e projetos.
